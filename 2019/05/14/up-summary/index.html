UP 上线审批系统总结

发表于 2019-05-14

UP 上线审批系统旨在替代原来的 jira，为公司上线提供规范化流程和为后续审计提供数据支持。作为首次全栈项目，花了不少的心思，同时也踩了不少的坑，本文就做个简单的总结。

项目主要技术栈

  • 前端

    • 前端框架: react
    • 状态管理: react-redux、redux-thunk、immutable
    • UI 框架: antd
    • http 请求: axios

  • 后端

    • web框架: flask
    • 数据库和 orm: mysql sqlalchemy
    • 异步任务: celery redis
    • 部署: uwsgi nginx supervisor

前端采坑

  • 使用create-react-app创建的项目，使用yarn eject命令将webpack配置暴露出来，以自定义antd主题等样式。

  • 安装less-loader，编辑config/webpack.conf.js自定义antd的样式。

  • 编辑config/webpack.conf.js按需加载antd。

  • 使用withRouter装饰器为组件提供location属性
    当某个组件没有直接挂在react-router-dom.Route下面时，props是没有location属性的，不能使用路由功能。
    此时需经过装饰器withRouter装饰后才具备路由功能。

  • 将 reducer 拆分后如何获取全局的 state

后端采坑

  • flask 在开发模式下会其两个进程，此时有可能引起循环导入。

  • view 函数中可返回string、tuple，那么框架是如何处理的，有什么要求

  • 修改 supervisor 的配置文件后，使用update仅更新修改的，而不是reload重新加载所有

  • nginx 路由匹配少写了/导致301

  • 当 property 遇上了 hasattr

  • werkzeug.secure_filename 不支持中文名文件
    主要的代码为filename = normalize("NFKD", filename).encode("ascii", "ignore")
    查看werkzeug.secure_filename的源码，重新实现了一个unicode版本的。

  • celery 占用 CPU 100%
    原因是 celery 的 worker 工作一段时间异常后未能成功退出和重启
    加入--max-tasks-per-child 20，每个 worker 执行 20 个任务后就退出重启。
    加入--time-limit 60，每个 worker 任务超时时间为 60s。
  • 配合 uwsgi 在多 worker 模式下抛出数据库连接异常问题
    单 worker 下测试正常，多 worker 会抛出 pymysql.err.OperationalError: (2013, 'Lost connection to MySQL server during query')
    原因是多 worker 下， uwsgi 会先启动一个 master 然后 fork 出其他的 worker，这些 worker 共享了同一个数据库连接，解决的办法是 uwsgi 添加--lazy-apps选项
