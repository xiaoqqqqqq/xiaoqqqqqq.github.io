Hello my blog

发表于 2017-07-27 | 分类于 开发工具

好记性不如烂笔头，从小学老师就教育我们学习要做笔记，要时常复习。对于走在技术的道路上的开发者，博客就是我们的笔记本，键盘就是我们的笔，把自己所学的知识用键盘细化成文字，记录于博客之中，时常复习才能进一步掌握。几经折腾，挑来选去，绕进了选择工具坑里跳出后，最终选择了Hexo+Next+Github Pages这个方案。以下为配置安装过的所记录的笔记。

环境的安装配置

nodejs的环境配置

  1. 安装nodejs 我这里是选择直接下载二进制包安装，安装过程除了更改安装路径其他的一路下一步，中途也没有遇到什么问题。
  2. 配置npm源，npm为nodejs的包管理器，安装nodejs后就自带有了，就像Python的pip，其默认安装源是在国外，下载速度不是一般的慢，在用户的家目录下C:\Users\username，创建.npmrc，加入淘宝源地址。

安装git

git也是直接下载二进制安装包，其安装包有三十多M，因为git还附带了许多工具，比如vim、grep、bash，利用这些工具再配合cmder后Windows下命令行不再值得吐槽了，后面有必要再开一篇记录下。

hexo 常用命令 以及部署运行

常用命令

  • init 新建项目
  • new 新建文章、标签页、分类页
  • generate 可简写为hexo g，根据source和theme生成静态文件到public中去。
  • publish 发布草稿，创作时可以在新建文章的时候指定为草稿类型，完成后在发布。
  • server 可简写为hexo s，启动本地服务器，可指定端口-p，和网卡-i，记录日志-l。
  • deploy 可简写为hexo d，根据项目配置文件中的deploy部分部署到远程服务器中去。
  • 官方文档

踩坑

跟着hexo文档和next文档 一路配置安装下来还是比较顺利的，过程遇到的两个主要的问题有：

  • 标签与分类数量显示不正确
    使用hexo clean后重新生成运行仍然不正确，部署到GitHub Pages确是正常的，这个可以直接忽略。

  • 发布到GitHub不成功
    首先尝试是否能直接clone，若clone git@github.com:xiaoq1024/xiaoq1024.github.io.git，若不能clone成功，检查网络是否畅通，部署公钥是否已经配置；若能clone成功，则尝试添加文件是否能推送是否具有写权限。

  • 代码里放django模板标签的时候报Template render error，查阅 https://hexo.io/docs/troubleshooting.html 得知，markdown文件里有不能识别的字符，排查后发现是django里面的过滤器和模板标签与bash的管道符号，不能被识别，特殊字符必须做转义处理，特殊字符及其转义表。
