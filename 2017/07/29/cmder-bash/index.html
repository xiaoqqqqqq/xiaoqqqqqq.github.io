cmder和bash真乃天作之合

发表于 2017-07-29 | 分类于 开发工具

在Windows下开发，对比Linux，Windows的命令行工具确实暗淡无色，或许有人要说PowerShell不服了，但熟悉Linux的命令操作后，我希望能保持与Linux一致的操作体验。钱包充足的可以考虑转OS X，但上了Windows的贼船，想中途下船确实不易，而且许多快捷键辅助工具都用习惯了，切换到OS X不一定能找到替代品，曾经也尝试过黑苹果，但是在是太折腾了。直接转Ubuntu？还是算了，一个autohotkey就让我选择放弃了。然而别的不敢说，Windows的软件数量肯定是完胜其他的操作系统的，微软没有做好的事情，在民间大牛不断地开发积累过程中产生了许多的cmd代替方案。其中我有尝试过babun和Git for Windows，最终选择了后者，各有优劣，前者也是灰常不错的。新开博客想记录一下配置过程，又重新去配置了一番，以下就是配置过程的记录，希望能给予苦苦挣扎在Windows的cmd中的开发者一点参考。

cmder

Portable console emulator for Windows

cmder在Github 上，有11K的星，看提交记录也是近几天的，项目很是活跃。进入官网看到的第一眼就是cmder的一张效果图，好清新有木有，完虐windows黑乎乎的cmd有木有。从他的slogan也可以看出，cmder其实只是一个命令行工具。

安装

cmder提供了两个版本下载。

  • mini版 不带Git for Windows，后面自己单独安装，个人更偏爱这个版本。
  • full版 带有Git for Windows，不需另外再安装Git for Windows。

定制

打开cmder，可以看到其默认配置呈现出来的效果并没有官网那么清新，当前每个人的审美标准不一样，但对于挑剔的我来说，进一步配置是很有必有的，个人比较喜欢像官网那样的效果：有最大的视角，杂七杂八没用的全部隐藏。Win+Alt+p呼出配置界面。

  • main

    • Appearance
      • 去掉 Show buttons(toobar) in tab tar 工具栏对我没什么用
      • 去掉 Show search filed in tab bar 去掉搜索框
    • Tab bar
      • 去除Tabs on bottom的勾选 将tab标签由底部移动到顶部
      • 隐藏Tab bar 勾选 Don’t Show 这个比较极端了 个人喜好
    • Update
      • 去掉更新检查，默认是Startup，去掉勾选就可以了

  • Startup

    • Environment
      中文乱码问题，初始情况下ls输出的文件中包含中文的结果会似这样显示成这样”‘$’\343\200\214\3..0\217\234\345\215\225‘@”。添加set LANG=zh_CN.UTF-8到Environment，重启即可显示正常的中文文件名了。前面的乱码就能成功显示”「开始」菜单@”。

  • Features

    • Text cursor
      • 用惯了Vim，这里将光标由竖线改为矩形，选择Rectangle，去掉Blinking闪烁。
    • Transparency
      • 默认情况下cmder已经给背景加了一定的透明度了，可以这里根据个人喜好再适当调节。

  • 修改cmder的命令提示符

    默认情况下cmder使用λ最为命令提示符，用惯bash的可能更习惯使用$，在配置界面没有找到修改这个的选项，于是乎用sublime在cmder的项目文件夹中搜索λ，定位到vendor/clink.lua

其实修改也没关系，因为后面主要是使用bash的，并不会用直接用cmder的命令行。

备份配置文件

配置好cmder后，应该将配置文件保存起来，在重置系统或迁移到新电脑时直接导入以前配置好的配置文件就可以了。
其实在配置面板中，右边的第一行Storge: D:\cmder_mini\vendor\conemu-maximus5\ConEmu.xml点导出即可，我更喜欢直接找到那个文件copy一份。

bash

babun

a Windows shell you will love!

在说bash之前不得不先提一下babun, babun把各种工具都打包好了，并且配置zsh，还给你安装好了oh-my-zsh，里面的工具软件集比后面介绍的Git for Windows要齐全很多。babun的命令行工具使用的是mintty，mintty这个软件跟cmder的性质差不多，总之babun就是开箱即用，灰常不错，我也使用过一段时间，但里面的目录与Windows有些不一致，比如用户的Home目录，并不是真实映射到Windows的用户Home目录的，而是内部自己维护了一个Home目录，部分软件在里面执行没有响应，比如：mysql、nodejs，netstat -ant | grep 3306 这样的命令也不能使用等等，最终也就没使用这个babun，然而这却是懒人必备，强烈建议试用一下，也许这就合适你，那么就不用再各种配置了。

Git for Windows

Git for Windows focuses on offering a lightweight, native set of tools that bring the full feature set of the Git SCM to Windows while providing appropriate user interfaces for experienced Git users and novices alike.

其实bash只是Git for Windows工具集中的其中一个软件而已，它提供了一个shell环境，供调用其他的工具。这个工具集附带了许多linux的软件工具，比如sed、grep、vim…。

下载安装，将安装目录d:/git/bin`配置到环境变量，即可在命令中直接呼叫bash

定制bash

首先切换到用户的家目录下，直接cd不带任何目录参数就可以直接切换到家目录下了，可以使用vim或者记事本等创建.bashrc文件，用于保存自己的bash自定义配置。

  • 提示符
    默认情况下bash的提示符是两行，第一行包含了用户名和主机，第二行就一个$，可以使用echo $PS1查看当前使用的提示符，默认的这个提示符，自我感觉太啰嗦了，用户名跟主机在个人电脑上用处不大，更没必要分两行。

这里模仿babun中zsh的提示符，在.bashrc加入export PS1='\[\033[33m\]{ \W }\[\033[32m\]__git_ps1\[\033[1;33m\] $ \[\033[0m\]'

关于bash的提示符以及颜色控制，可以参考TLCL这个章节

  • 命令别名

    命令别名alias是bash的特性之一，可以给常用的命令其个别名，非常好用。

  • 命令历史

    • 自动保存历史命令
      在windows下使用bash，有时候习惯使用Ctrl+w来关闭bash，导致会话的历史命令没能成功追加到命令历史文件中，Google得到的方案是修改PROMRT_COMMAND，同样在.bashrc加入：export PROMPT_COMMAND='history -a'。这样每敲一条命令，就会自动追加到命令历史文件中。

    • 修改历史命令的记录方式
      对于保存命令的策略，bash提供了几种方案（这里让我想起了安卓Activity的启动方式），具体的方案异同就不展开了，可以通过修改HISTCONTROL来定制。在默认情况下，不像Linux，在Windows下的bash这个变量未创建，导致重复执行的命令都会保存到历史命令中，这并不是我所希望的。

  • 环境变量

    在Windows中安装一个新的开发软件后，如果安装时没有自动添加到环境变量，往往需要我们手工点到环境变量里面添加，现在使用bash后不用那么麻烦了，直接在.bashrc中管理你的环境变量。其实还可以使用ln创建软连接到/usr/bin中，但是这种方式感觉不太稳定，还是比较推荐使用环境变量。

cmder与bash的结合

默认情况下，Git for Windwos自带有命令行工具D:\git\git-bash.exe，但是比起cmder的界面实在是差太大了而且也不支持多tab，但是他自带的软件工具集厉害呀。相反地，cmder界面操作友好，支持多tab，但自身的工具集是不及Git for Windows的。这岂不是天作之合？赶紧的，在一起！在一起！在一起！

  • 将bash配置到cmder的task中

    win+alt+p调出配置面板， 在Startup Tasks里面默认配置有8个，第7个第8个默认是调用cmder full安装下的bash，这里修改为我自己的安装路径 *D:\git\bin\bash.exe -new_console:d:%USERPROFILE% D:\git\bin\bash.exe -new_console:d:%USERPROFILE%，再将这两个移到第1第2位，shift+alt+1,shift+alt+2即可看到效果。

  • cmder启动时就开启bash

    默认情况下，打开cmder进入的是cmder的命令行工具，这里修改为打开cmder时就进入bash。
    在配置面板中选中Startup –> Sepcified named task，然后选择前面创建的bash task就可以了。

总结

前文介绍了三种工具: cmder、Git for Windows、babun，正所谓尺有所短，寸有所长，没有哪一个是完美的，挑选符合自己的工具才是正道，babun集齐了命令行工具、各种软件工具、zsh；cmder则更偏向做好命令行；而Git for Windows则偏向提供软件工具集。总之babun或是cmder搭配Git for Windows都能在Windows享受良好的命令行操作体验，对于习惯Linux命令行和喜欢命令行而不是文件管理的开发者提供了极大的便利。稍稍折腾配置一番，无需再吐槽Windows黑乎乎的cmd了，也无需羡慕苹果用户又有qq又有命令行了，为何而不为？
