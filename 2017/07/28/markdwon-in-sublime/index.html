Sublime下配置Markdown写作环境

发表于 2017-07-28 | 分类于 开发工具

在Window写Markdown文件，可以使用像MarkdownPad这样专业的软件，使用双屏，左边写作，右边可以即使预览效果，确实是很方便，但是偏爱Sublime，又离不开Vim，只能在Sublime下在折腾一番了。
以下为配置过程整理的笔记，主要参考了这篇文章和对应插件的wiki。

MarkdownEditing

Powerful Markdown package for Sublime Text with better syntax understanding and good color schemes.

支持Markdown语法高亮，带有三种Markdown风格：标准、GitHub Flavored Markdown、MultiMarkdown。本文主要使用第二种GFM。

定制配置

在Preferencs的Package Setting下选择MarkdownEditing，先从GFW Settings-Default中全选，复制到GFW Settings-User

快捷键

- Ctrl+Win+v 生成超链接，如果剪切板里的内容就是一个超链接地址，则会自动填充。
- Alt+b Alt+i 粗体，斜体
- Shift+Tab 折叠/打开当前章节
- Ctrl+Shift+ PageUp/PageDown 在章节间跳转

MarkdownPreview

markdown preview and build plugin for sublime text 2/3

将Markdown文件”编译”成html文件，方便在浏览器中查看效果，浏览器了一下其默认配置，没有什么需要特别修改定制的。
build快捷键在Sublime里默认为Ctrl+B，由于与我使用了的Vim插件插件有冲突，修改为F5，编译后可在与当前Markdown统一路径下生成html文件，拖入浏览器即可浏览效果。

总结

利用这两款插件基本上已经能满足Markdown写作需求了，另外还可以使用对主题配色方案等进一步定制，但自带的黑色主题能已经满足我的需要了，就不进一步折腾了。
对比MarkdonwPad还是有些不足的，但必须有所取舍，就像ide与文本编辑器之间作选择吧，起码我目前还是选择站队Sublime。
